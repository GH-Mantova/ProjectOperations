The full grammar (design-doc summary):
Conditions per field type — choice/multi: is · is not · contains any/all/none of · number selected ≥/≤ • text: filled · empty · contains · doesn't contain • numbers/dates: = ≠ > < between · before/after/within • tables & repeating sections: has any entry where… · has no entry where… · entry count · column total • system values: compare against asset readings, worker competencies, site attributes, weather, the filler's role. Groups combine with ALL (and) / ANY (or), nestable.
Actions — show/hide/require field·section·page • set a value • filter another field's options • WARN (acknowledge to continue, acknowledgement recorded) • BLOCK (hard stop with your custom message) • alert a named person or role via in-app/email/Teams, message templated with live answers • change the approval chain • push actions into modules (flag asset, raise maintenance, log hazard) • change the after-submit message.
Timing — each rule chooses: live while filling · on submit · on approval.
