# 1. Platform Foundation

## Purpose

The base layer everything else stands on ‚Äî health checks, error envelope,
exception filter, logging, paginated list shape, OpenAPI bootstrap, runtime
config, telemetry hooks. Operations / DevOps facing ‚Äî users never see it
directly, but every other module breaks if this layer is off.

## Surface area

**Routes (frontend):**
- `/` ‚Äî `DashboardPlaceholderPage` (the operations overview that mounts when
  the auth layer hands off to the shell)
- `/login` ‚Äî the auth boundary
- Catch-all `*` ‚Üí currently navigates to `/` (PHASE 6 tech-debt item:
  replace with proper 404)
- ShellLayout ‚Äî top bar with breadcrumb, notifications bell, ‚åòK search,
  user avatar; 240px / 64px collapsed sidebar; bottom tab bar < 768px

**API endpoints (key):**
- `GET /api/v1/health` ‚Äî returns OK + version + DB connectivity
- Swagger UI: `GET /api/docs`
- Standard list envelope: `{ items, total, page, pageSize }`
- Standard error envelope from the global exception filter
- `GET/PATCH /api/v1/notifications`, `PATCH /api/v1/notifications/read-all`

**DB entities:**
- `HealthcheckSeedMarker` ‚Äî confirms seed ran
- `Notification` ‚Äî bell-icon items
- `AuditLog` ‚Äî generated by AuditService for every write
- `RefreshToken`

## What to watch for during sanity check

- **Error boundary fallback copy** ‚Äî PR #203 gave each JobDetailPage tab
  its own boundary. If a section throws, you should see "Something went
  wrong" inline, not a blank page. The B01.1 fix is the regression risk;
  break a tab on purpose to confirm.
- **Bundle splitting** ‚Äî PR #312 introduced `vite.config.ts`
  `manualChunks`. Watch for missing JS during route changes (lazy loaders
  that 404 in dev or fail in production).
- **Notifications bell live update** ‚Äî recent notifications should appear
  without page refresh. Check the WebSocket / polling cadence.
- **Service worker update strategy** ‚Äî P-platform3 in the Design Map is
  open (Approach A skip-waiting + toast, deferred). If a stale SW survives
  a deploy, the symptom is "old UI, new backend". Hard refresh as a
  fallback.
- **OpenAPI completeness** ‚Äî every recent PR (#22, #44, #56, #281, #296,
  #308) added Swagger decorators. Verify the controllers Marco cares
  about (`tenders`, `jobs`, `maintenance`, `scheduler`, `forms`,
  `master-data`, `tender-entries`) all have full request/response docs.
- **Catch-all bouncing** ‚Äî unknown URL silently rewriting to `/` masks
  bugs (see PR #120 audit note). If a deep link 404s in the UI, the
  user has zero diagnostic.

## Edge cases worth probing

- **Empty notifications panel** ‚Äî does an empty state render or a blank?
- **100+ notifications** ‚Äî does the panel scroll or break?
- **Permission-gated nav items** ‚Äî log in as a non-admin and confirm
  hidden groups stay hidden (sidebar role gating)
- **Mobile width (375px)** ‚Äî bottom tab bar visible, top bar shrinks,
  no horizontal scrollbar
- **Network failure mid-action** ‚Äî kill the API, click anything, check
  the user-facing error is sane (no raw stack traces)
- **Concurrent edit** ‚Äî two tabs editing the same record, confirm the
  server returns a 409 or last-write-wins without silent corruption
- **Service worker stale cache** ‚Äî install PWA, deploy a UI change, see
  if user is prompted to reload (today: NetworkFirst 24h cache, PHASE
  6 tech debt item M14)
- **Browser console** ‚Äî should be clean. Any uncaught errors / 404s on
  asset requests = P1 finding
